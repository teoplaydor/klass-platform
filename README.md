# Класс — учебная платформа (white-label)

Российский аналог Google Classroom для негосударственного образования:
частные и онлайн-школы, учебные центры, СПО/ДПО, вузы, корпоративное обучение.
Продукт спроектирован для передачи покупателю: брендинг и комплектация меняются
**одним конфигурационным файлом**, без правки исходного кода.

## Возможности

- **Курсы**: создание, цветовые темы, код приглашения, копирование, архив.
- **Лента**: объявления с вложениями, закрепление, комментарии, отложенная
  публикация, три режима участия учеников.
- **Задания** четырёх типов: задание, **тест с автопроверкой**, вопрос, материал;
  темы-разделы, сроки, баллы, черновики, публикация по расписанию,
  назначение отдельным ученикам.
- **Сдачи**: жизненный цикл «назначено → сдано → проверено → возвращено»,
  пересдача, история событий, статус «с опозданием».
- **Проверка работ**: список учеников со статусами, листание, черновик оценки,
  возврат с оценкой, приватные комментарии, банк комментариев учителя,
  критериальные рубрики.
- **Журнал оценок**: таблица «ученики × задания», среднее по классу,
  шкалы отображения: 5-балльная (дефолт), баллы, проценты.
- **Агрегаторы**: «Список дел» ученика, «На проверку» учителя, календарь дедлайнов.
- **Уведомления**: внутренние («колокольчик») + точка расширения для
  email/Telegram/SMS-каналов.
- **Файлы**: загрузка с контролем прав доступа на скачивание.

## Быстрый старт

Требования: Node.js ≥ 22.5 (рекомендуется 24+). Внешние сервисы не нужны —
БД встроена, шрифты системные, внешних CDN нет.

```bash
npm install          # зависимости
npm run seed         # демо-данные (учитель, ученики, курсы)
npm run dev          # API на :3000, веб на :5173
```

Демо-доступы: `teacher@demo.ru`, `student1@demo.ru`, `admin@demo.ru` — пароль `demo1234`.

Продакшен-сборка (один процесс — сервер отдаёт и API, и фронтенд):

```bash
npm run build
npm start            # http://localhost:3000
```

## Структура

```
brand.config.json    ← ЕДИНАЯ точка кастомизации покупателем
server/              ← API: Express 5 + встроенный SQLite (node:sqlite)
  src/core/          ← БД, схема, доступы, ошибки, планировщик
  src/modules/       ← модули по доменам (auth, courses, coursework, …)
web/                 ← SPA: React 18 + Vite, дизайн-токены на CSS-переменных
docs/                ← документация пакета передачи + результаты ресёрча
```

## Ключевые принципы

1. **Configuration as data**: бренд, фичефлаги и лимиты — в `brand.config.json`;
   фронтенд получает конфиг с сервера при загрузке, поэтому ребрендинг
   не требует пересборки.
2. **Ноль внешних зависимостей в рантайме**: никаких Google Fonts, CDN,
   внешней аналитики — продукт работает в изолированном контуре.
3. **Модульный монолит**: каждый модуль — самостоятельный Router со своей
   зоной данных; состав приложения собирается в одном файле `server/src/index.ts`.
4. **Переносимая схема данных** по модели Google Classroom API
   (Course / CourseWork / StudentSubmission / Rubric) — путь миграции
   на PostgreSQL описан в docs/ИНТЕГРАЦИЯ.md.
