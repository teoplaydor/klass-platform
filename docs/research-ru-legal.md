# Отчёт исследования: ru-legal

Регуляторная среда для образовательного ПО в РФ (2025–2026) жёстко делит рынок на два контура. Закрытый контур — государственные и муниципальные школы и колледжи: по ч. 3.1 ст. 16 закона 273-ФЗ с 01.01.2023 при реализации основных общеобразовательных программ и программ СПО с применением электронного обучения/ДОТ, предполагающих обработку ПДн обучающихся, организация обязана использовать только государственные информационные системы (ФГИС «Моя школа» по ПП № 1241, региональные ГИС) и многофункциональный сервис обмена информацией (национальный мессенджер MAX, заменивший Сферум к 01.11.2025). Норма распространяется на все организации, ведущие образовательную деятельность по этим программам, включая частные аккредитованные школы, поэтому продавать частное white-label ПО в этот сегмент как основную платформу учёта/обучения нельзя — только как дополнительные сервисы вне контура обработки ПДн по основным программам. Открытый контур, куда реально продавать частное ПО: онлайн-школы и ДПО, дополнительное образование детей и взрослых, корпоративное обучение (вообще не лицензируется как образование, если без выдачи документов), вузы (требований об исключительности ГИС нет, но госвузы при закупках обязаны брать ПО из реестра Минцифры по ПП № 1236, действует политика импортозамещения Минобрнауки). При этом ПП № 1678 (действует с 01.09.2024) требует от всех образовательных организаций при ЭО/ДОТ использовать ПО из единого реестра российских программ, а онлайн-школы клиента должны иметь образовательную лицензию (с 01.03.2025 — ещё и аккредитацию программ), что делает попадание продукта в реестр Минцифры фактически обязательным условием продаж в лицензированный EdTech.

По персональным данным критичны три блока. Локализация: с 01.07.2025 первичный сбор и запись ПДн граждан РФ допускаются только в базах данных на территории РФ (штраф до 6 млн руб., повторно до 18 млн), а с 01.02.2024 хостинг в РФ вправе оказывать только провайдеры из реестра Роскомнадзора (инфраструктура в РФ, российские владельцы, взаимодействие с ГосСОПКА). Согласия: с 01.09.2025 согласие на обработку ПДн — отдельный документ (не пункт оферты и не предотмеченная галочка) с конкретными целями и перечнем данных; для несовершеннолетних до 18 лет практика РКН требует согласие законного представителя, отдельное на каждого ребёнка, плюс отдельные согласия на фото/публикации; вендор платформы выступает обработчиком по договору поручения (ст. 6 ч. 3 152-ФЗ) с каждой школой/клиентом. Ответственность: с 30.05.2025 штрафы за утечки 3–20 млн руб., повторно оборотные 1–3% выручки (20–500 млн), уведомление РКН об инциденте — 24 часа, расследование — 72 часа, уведомление о начале обработки обязательно для всех операторов (штраф 100–300 тыс.), введена уголовная ответственность по ст. 272.1 УК РФ.

Для включения в реестр отечественного ПО (ПП № 1236) от white-label продукта требуется: исключительные права целиком у российского юрлица с долей иностранного участия <50% (лицензии недостаточно — нужен договор отчуждения, что и есть суть white-label сделки); выплаты иностранным правообладателям/подрядчикам <30% выручки от продукта; отсутствие санкционных компонентов и платных иностранных зависимостей (платные Oracle/MySQL — типовая причина отказа; нужен переход на PostgreSQL/Postgres Pro); open source допустим при лицензиях без территориальных ограничений и без препятствий исключительному праву; полный перечень всех компонентов с лицензиями (SBOM); техдокументация по ГОСТ на русском; серверы SaaS-версии — только в РФ; продукт должен реально продаваться. С 01.03.2026 поэтапно вводится «правило двух ОС» (совместимость минимум с двумя ОС из реестра — Astra Linux, РЕД ОС, ALT, ROSA): офисное ПО — с 01.09.2026, системное — с 01.01.2027, прикладное (куда попадает LMS) — с 01.06.2027. Аутентификация: по 406-ФЗ с 01.12.2023 авторизация на российских ресурсах допустима только через российский номер телефона, ЕСИА, ЕБС или российскую систему (VK ID, Яндекс ID); Google/Apple ID запрещены, в июне 2026 приняты штрафы до 700 тыс. руб. для юрлиц; прямой доступ коммерческих организаций к ЕСИА ограничен закрытым перечнем категорий (банки, страховые, операторы связи), поэтому для EdTech базовый стек — телефон/email + VK ID/Яндекс ID с архитектурной готовностью к ЕСИА. Сертификация ФСТЭК самому прикладному ПО не нужна, если оно не выполняет функции защиты информации и не поставляется в состав ГИС;但 при поставке в ГИС система подлежит обязательной аттестации (приказ № 17, с 01.03.2026 заменяется приказом № 117) с использованием сертифицированных СЗИ, а для коммерческих ИСПДн действует приказ ФСТЭК № 21 — продукт должен поддерживать меры защиты по уровням защищённости УЗ1–УЗ4 (ПП № 1119; данные детей и сведения о здоровье повышают уровень).

## Факты
- 152-ФЗ, локализация: с 1 июля 2025 первичный сбор и запись ПДн граждан РФ допускаются только в базах данных на территории РФ; зарубежные сервисы могут использоваться лишь как вторичный слой при соблюдении правил трансграничной передачи; штраф за нарушение локализации — до 6 млн руб., повторно до 18 млн (ст. 13.11 ч. 8 КоАП).
- 152-ФЗ, штрафы с 30 мая 2025: утечка ПДн — 3–20 млн руб. в зависимости от числа субъектов, повторная утечка — оборотный штраф 1–3% годовой выручки (мин. 20 млн, макс. 500 млн руб.); уведомление РКН об утечке — в течение 24 часов, внутреннее расследование — 72 часа.
- Все операторы ПДн обязаны подать уведомление о начале обработки в Роскомнадзор; неуведомление — штраф 100–300 тыс. руб.; с 11.12.2024 действует уголовная ответственность за незаконный оборот ПДн (ст. 272.1 УК РФ, до 10 лет).
- Согласия с 1 сентября 2025: согласие на обработку ПДн должно быть отдельным самостоятельным документом (не пунктом договора/оферты, без предотмеченных галочек), с конкретным перечнем данных, целями по группам, сроком действия и процедурой отзыва.
- ПДн несовершеннолетних: по практике проверок РКН согласие законного представителя требуется для всех детей до 18 лет; оформляется отдельное согласие на каждого ребёнка и отдельные согласия на фото/видео и публикации; данные о здоровье — специальная категория (штраф за обработку без основания до 1,5 млн руб.).
- В схеме «школа/онлайн-школа — платформа» образовательная организация остаётся оператором ПДн, а вендор платформы — обработчиком: с каждым клиентом нужен договор поручения обработки ПДн (ч. 3 ст. 6 152-ФЗ) с перечнем мер защиты — РКН проверяет наличие таких договоров у школ.
- Реестр отечественного ПО (ПП № 1236): исключительные права на продукт должны целиком принадлежать российскому юрлицу с долей иностранного участия менее 50% (или гражданам РФ); для white-label это означает обязательное полное отчуждение исключительных прав покупателю — лицензионной модели для реестра недостаточно.
- Реестр ПО: сумма выплат иностранным правообладателям и подрядчикам (лицензии, разработка) должна быть менее 30% выручки от реализации ПО за год — постоянные роялти зарубежному white-label поставщику ставят включение в реестр под угрозу.
- Реестр ПО, кодовая база: запрещены компоненты под санкционными ограничениями и критическая зависимость от платных иностранных компонентов (платные Oracle/MySQL — типовая причина отказа); требуется полный перечень всех внешних компонентов с указанием правообладателей и лицензий (SBOM); open source допустим, если лицензия не препятствует исключительному праву и не содержит территориальных ограничений для РФ.
- Реестр ПО: техническая документация по ГОСТ Р ИСО/МЭК 12207-2010 на русском (установка, функциональные характеристики, жизненный цикл), техподдержка и модернизация силами российского юрлица, серверная инфраструктура SaaS-версии — только на территории РФ, продукт должен свободно продаваться; экспертиза занимает до 45 рабочих дней.
- «Правило двух ОС»: постановлением Правительства (вступает в силу 01.03.2026) для реестра вводится обязательная совместимость минимум с двумя российскими ОС из реестра (Astra Linux, РЕД ОС, ALT, ROSA): офисное ПО — с 01.09.2026, системное (виртуализация, облака, СУБД, мониторинг) — с 01.01.2027, прикладное ПО (включая LMS) и средства защиты — с 01.06.2027, промышленное — с 01.01.2028.
- Практический технологический стек для соответствия реестру: серверная часть должна работать на Astra Linux и/или РЕД ОС, СУБД — PostgreSQL/Postgres Pro (свободная или реестровая), без завязки на проприетарные иностранные облачные API (AWS, Google, Firebase) и без управления обновлениями из-за рубежа.
- Госшколы — закрытый сегмент: по ч. 3.1 ст. 16 273-ФЗ с 01.01.2023 при реализации основных общеобразовательных программ и программ СПО с применением ЭО/ДОТ, предусматривающих обработку ПДн обучающихся, организация обязана использовать государственные информационные системы и многофункциональный сервис обмена информацией; норма касается всех организаций, ведущих такие программы, включая частные аккредитованные школы.
- ФГИС «Моя школа» (ПП № 1241 от 13.07.2022) — обязательная федеральная система для общего образования (электронные журналы/дневники, образовательный контент); вход пользователей только через подтверждённую учётную запись ЕСИА (Госуслуги); выбор конкретных ГИС (федеральная или региональная) — за органами власти субъектов РФ.
- Школьные коммуникации переведены со Сферума на национальный мессенджер MAX (закон от 24.06.2025 о многофункциональном сервисе обмена информацией); переход системы образования завершён к 01.11.2025 — частные мессенджеры/видеосвязь в госшколах вне правового поля.
- ПП № 1678 от 11.10.2023 (действует с 01.09.2024): все организации с лицензией на образовательную деятельность (включая частные онлайн-школы и ДПО) при применении ЭО/ДОТ обязаны использовать ПО из единого реестра российских программ либо собственные разработки; обязательна идентификация/аутентификация обучающихся при аттестации; профессиональные программы и ДПО можно вести полностью дистанционно.
- Контент для аккредитованных школьных программ: допускаются только электронные образовательные ресурсы из федерального перечня ЭОР (приказ Минпросвещения № 551 от 23.07.2025, порядок формирования — приказ № 1 от 09.01.2025); включение — через ежегодную экспертизу Минпросвещения (окно подачи весной) на соответствие ФГОС и федеральным образовательным программам.
- Доступные сегменты для частного white-label ПО: онлайн-школы и курсы (доп. образование детей и взрослых, ДПО), корпоративное обучение (не лицензируется, если без итоговой аттестации и выдачи документов об образовании), вузы (нет требования исключительности ГИС; госвузы при закупках обязаны выбирать ПО из реестра Минцифры по запрету ПП № 1236, Минобрнауки ведёт политику импортозамещения), репетиторские и тренинговые форматы.
- Клиенты-онлайн-школы обязаны иметь лицензию на образовательную деятельность (в 2025–2026 контроль ужесточён, госпошлина отменена до 2029), а с 01.03.2025 — проходить госаккредитацию программ; вендор платформы лицензию не получает, но платформа должна поддерживать требования лицензиатов: структура программ, учёт обучающихся, идентификация при аттестации, фиксация результатов.
- Аутентификация (406-ФЗ, с 01.12.2023): авторизация пользователей на российских сайтах и приложениях допустима только четырьмя способами — по российскому номеру телефона, через ЕСИА (Госуслуги), через Единую биометрическую систему или через российскую систему авторизации (VK ID, Яндекс ID и т.п.); вход через Google/Apple ID запрещён; в июне 2026 Госдума приняла штрафы за нарушение — до 500–700 тыс. руб. для юрлиц.
- Прямое подключение к ЕСИА доступно госорганизациям и лишь закрытому перечню коммерческих категорий (банки, страховые, НПФ, операторы связи и т.п.) по протоколу OpenID Connect; обычный EdTech-сервис подключить ЕСИА напрямую не может — рабочий стек: телефон/email + VK ID/Яндекс ID, с архитектурной готовностью к ЕСИА (OIDC/SAML) для интеграций с госсистемами; для госсайтов ЕСИА обязательна с 30.03.2025.
- Хостинг: с 01.02.2024 услуги хостинга в РФ вправе оказывать только провайдеры из реестра Роскомнадзора (инфраструктура на территории РФ, российские учредители, взаимодействие с ГосСОПКА, сообщение об инцидентах в течение 24 часов); продукт должен развёртываться у реестрового провайдера, а БД с ПДн — физически находиться в РФ.
- ФСТЭК: сертификация обязательна только для средств защиты информации; прикладное ПО (LMS) сертифицируется добровольно по уровням доверия; но если продукт поставляется в составе ГИС (региональные образовательные системы), такая система подлежит обязательной аттестации по приказу ФСТЭК № 17 (с 01.03.2026 — новый приказ № 117) с использованием сертифицированных СЗИ.
- Для коммерческих ИСПДн действует приказ ФСТЭК № 21: оператор определяет уровень защищённости УЗ1–УЗ4 по ПП № 1119 и реализует соответствующие меры; обработка данных несовершеннолетних и сведений о здоровье повышает уровень; аттестация для коммерческих систем добровольна, но СЗИ должны пройти оценку соответствия — white-label платформа должна поддерживать журналирование, разграничение доступа и встраивание сертифицированных СЗИ.
- Критический чек-лист для white-label продукта: 1) полное отчуждение исключительных прав российскому покупателю и юридическая чистота кода для реестра Минцифры; 2) отсутствие санкционных и платных иностранных зависимостей, SBOM, миграция на PostgreSQL/Postgres Pro; 3) совместимость с Astra Linux/РЕД ОС (правило двух ОС к 01.06.2027 для прикладного ПО); 4) развёртывание только в РФ у хостера из реестра РКН, локализация ПДн; 5) авторизация без иностранных ID — телефон/email/VK ID/Яндекс ID + готовность к ЕСИА; 6) модуль управления согласиями (отдельные согласия, родительские согласия до 18 лет, отзыв и удаление данных, журнал согласий); 7) функции реагирования на инциденты под 24-часовое уведомление РКН; 8) русскоязычная документация по ГОСТ; 9) позиционирование на сегменты ДПО/онлайн-школ/корпоративного обучения/вузов, а не на государственные школы.
