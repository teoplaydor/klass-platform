# White-label: что меняет покупатель

**Принцип: ни одной правки исходного кода для типового внедрения.**
Вся кастомизация — два файла: `brand.config.json` (бренд и комплектация)
и переменные окружения (среда).

## 1. brand.config.json — единая точка брендирования

Файл лежит в корне проекта. Сервер читает его при старте; фронтенд получает
его по `GET /api/config` при загрузке страницы — поэтому **ребрендинг не
требует пересборки фронтенда**, достаточно перезапустить сервер.

```jsonc
{
  "product": {
    "name": "Класс",            // название продукта: шапка, <title>, страница входа
    "shortName": "Класс",       // короткое имя (мобильные сценарии)
    "tagline": "Платформа …",   // подзаголовок на странице входа
    "company": "Ваша компания", // юридическое имя владельца
    "supportEmail": "…",        // контакт поддержки
    "logoText": "К"             // монограмма-логотип (1–2 символа)
  },
  "theme": {
    "colorPrimary": "#2456a4",      // основной цвет (кнопки, активные вкладки)
    "colorPrimaryHover": "#1d4789",
    "colorAccent": "#0e7a5f",       // позитивные статусы («проверено»)
    "colorDanger": "#b3261e",       // ошибки, «не сдано»
    "fontFamily": "system-ui, …",   // шрифт; только локальные/системные (без CDN!)
    "radius": "8px",                // скругление углов всего интерфейса
    "courseColors": { … }           // палитра обложек курсов (ключ: hex)
  },
  "features": { … },   // см. раздел 2
  "limits": {
    "maxUploadSizeMb": 50,          // лимит размера файла
    "maxAttachmentsPerPost": 10     // вложений на пост/сдачу
  }
}
```

Как это работает: `web/src/brand.tsx → applyBrand()` переносит токены
в CSS-переменные (`--brand-primary` и т. д.), на которых построен весь
`web/src/theme.css`. Трёхслойная схема токенов: примитивы бренда →
семантические цвета → компоненты.

### Чек-лист ребрендинга (15 минут)

1. Заменить блок `product` (название, монограмма, контакты).
2. Заменить цвета `theme` на фирменные.
3. При необходимости — палитру курсов `courseColors`.
4. Перезапустить сервер. Готово: интерфейс, заголовки вкладок, страница
   входа и уведомления используют новый бренд.

## 2. Фичефлаги — комплектация продукта

Блок `features` включает/отключает модули. Проверки выполняются и на сервере
(API вернёт 403), и в интерфейсе (элементы скрываются).

| Флаг | Что отключает |
| --- | --- |
| `registration` | самостоятельную регистрацию (останется вход) |
| `courseCreationByAnyone` | создание курсов не-администраторами |
| `grades` | вкладку и API журнала оценок |
| `rubrics` | критериальные рубрики |
| `quizzes` | конструктор тестов с автопроверкой |
| `calendar` | календарь дедлайнов |
| `todo` | агрегатор «Список дел» / «На проверку» |
| `notifications` | «колокольчик» уведомлений |
| `fileUploads` | загрузку файлов (ссылки останутся) |
| `comments` | все комментарии |
| `privateComments` | приватную переписку по сдаче |
| `scheduling` | отложенную публикацию |
| `archive` | архивирование курсов |

Сценарий: покупатель продаёт «лайт»-тариф без тестов и рубрик —
выключает два флага, это вся работа.

## 3. Переменные окружения (среда)

| Переменная | Назначение | Дефолт |
| --- | --- | --- |
| `PORT` | порт сервера | `3000` |
| `DATA_DIR` | каталог данных (БД, файлы, секрет) | `server/data` |
| `SESSION_SECRET` | секрет подписи сессий (обязателен в продакшене) | автогенерация в `DATA_DIR` |
| `SESSION_TTL_DAYS` | срок жизни сессии | `30` |
| `NODE_ENV` | `production` включает Secure-cookie | — |

## 4. Что НЕ нужно менять

- CSS-файлы: все цвета и радиусы приходят из токенов.
- Тексты интерфейса: написаны нейтрально, без упоминания бренда
  (название продукта везде подставляется из конфига).
- Код модулей: интеграции выполняются через точки расширения
  (см. МОДУЛИ.md и ИНТЕГРАЦИЯ.md), а не правкой ядра.

## 5. Иерархия кастомизации (по нарастанию сложности)

1. **Токены бренда** — без кода, минуты.
2. **Фичефлаги и лимиты** — без кода, минуты.
3. **Каналы уведомлений** (email/Telegram/SMS) — один класс-адаптер,
   регистрируется в `notifications/service.ts` (см. ИНТЕГРАЦИЯ.md).
4. **Провайдеры входа** (VK ID, Яндекс ID, ЕСИА) — добавление маршрута
   в модуль auth по шаблону из ИНТЕГРАЦИЯ.md.
5. **Новые модули** — папка + Router + строка в index.ts.

Форки под отдельных клиентов запрещены идеологией продукта: вся вариативность —
данные конфига, не ветки кода.
