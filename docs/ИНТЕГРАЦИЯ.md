# Интеграция и развёртывание

Документ для команды покупателя: как развернуть, перевести на PostgreSQL,
подключить российские сервисы и подготовиться к реестру отечественного ПО.

## 1. Развёртывание

### Разработка

```bash
npm install && npm run seed && npm run dev
```

### Продакшен (один процесс)

```bash
npm run build        # web/dist + server/dist
SESSION_SECRET=<случайные 64 hex> NODE_ENV=production npm start
```

Сервер отдаёт API и статику фронтенда на одном порту — за обратным прокси
(nginx) нужен только один upstream. Файлы и БД — в `DATA_DIR`
(по умолчанию `server/data`); этот каталог входит в резервное копирование.

Требования 152-ФЗ: хостинг в РФ у провайдера из реестра РКН; все
персональные данные хранятся только в `DATA_DIR` инсталляции — наружу
платформа ничего не отправляет (проверяется отсутствием исходящих запросов
в коде: `grep -r "fetch\|axios\|http.request" server/src`).

### systemd-юнит (пример)

```ini
[Service]
WorkingDirectory=/opt/klass
Environment=NODE_ENV=production
Environment=DATA_DIR=/var/lib/klass
EnvironmentFile=/etc/klass/env   ; SESSION_SECRET=…
ExecStart=/usr/bin/node --no-warnings server/dist/index.js
Restart=always
```

Совместимость с Astra Linux / РЕД ОС: продукт — чистый Node.js без нативных
модулей, работает на любом дистрибутиве с Node 22+ (включая сборки из
репозиториев отечественных ОС).

## 2. Миграция на PostgreSQL

Зачем: требование реестра отечественного ПО (совместимость с PostgreSQL /
Postgres Pro) и горизонтальное масштабирование.

Объём работы ограничен по построению — весь SQL проходит через один файл
`server/src/core/db.ts` (четыре функции: `get/all/run/tx`).

1. Заменить реализацию на `pg` (Pool): параметры `?` → `$1…$n`
   (механическая замена в db.ts — обёртка-транслятор ~10 строк).
2. `schema.sql`: `INTEGER PRIMARY KEY AUTOINCREMENT` → `BIGSERIAL PRIMARY KEY`;
   `INSERT OR IGNORE` → `ON CONFLICT DO NOTHING` (2 места);
   `ON CONFLICT(...) DO UPDATE` уже совместим.
3. `lastInsertRowid` → `RETURNING id` (в обёртке `run`).
4. Остальной код модулей не меняется: диалект-специфичных запросов нет.

Оценка: 1–2 дня работы + прогон сценариев.

## 3. Аутентификация: VK ID, Яндекс ID, ЕСИА

По 406-ФЗ авторизация в российских сервисах — только российские провайдеры
(Google/Apple ID запрещены). Подключение — новый маршрут в модуле auth:

```ts
// server/src/modules/auth/vkid.ts (схема)
authRouter.get('/vkid/callback', async (req, res) => {
  const profile = await exchangeCodeForProfile(req.query.code); // OAuth-обмен
  let user = get('SELECT id FROM users WHERE email = ?', profile.email);
  if (!user) { /* создать пользователя без пароля */ }
  setSessionCookie(res, user.id);   // существующее ядро сессий
  res.redirect('/');
});
```

Ядро сессий (`tokens.ts`, cookie) не меняется. ЕСИА — тот же паттерн
(OIDC), но прямой доступ коммерческим организациям ограничен — адаптер
готовится «на полку» для интеграций покупателя с госструктурами.

## 4. Каналы уведомлений (email, Telegram, SMS, MAX)

Интерфейс `NotificationChannel` (см. МОДУЛИ.md → notifications):

```ts
// server/src/modules/notifications/telegram.ts (пример)
import { registerChannel } from './service.js';

export function enableTelegram(botToken: string) {
  registerChannel({
    async deliver(msg) {
      const chatId = get('SELECT tg_chat_id FROM user_tg WHERE user_id = ?', msg.userId);
      if (!chatId) return;
      await fetch(`https://api.telegram.org/bot${botToken}/sendMessage`, { … });
    },
  });
}
```

Email-дайджесты родителям (вывод исследования: ключевая фича ру-рынка) —
тот же механизм + cron-выборка из `submissions`/`coursework` (схема готова:
все нужные статусы и сроки в БД).

## 5. Видеосвязь

Собственная ВКС не реализуется (вывод исследования: «не делать»). Поле
для ссылки на комнату (МТС Линк, SberJazz, Телемост, self-hosted Jitsi) —
добавить колонку `meet_url` в `courses`, поле в настройках курса и кнопку
в баннере. ~2 часа работы, шаблон правок очевиден из `CourseSettingsModal`.

## 7. Юридические модули (дорожная карта для покупателя)

Из исследования (research-ru-legal.md), к реализации по мере выхода на сегменты:

1. **Модуль согласий на обработку ПДн** (требование с 01.09.2025): отдельный
   документ согласия при регистрации, согласие законного представителя для
   несовершеннолетних, журнал согласий, отзыв и удаление данных.
   Схема расширения: таблица `consents(user_id, kind, granted_at, revoked_at, document_version)`.
2. **Аудит-лог администратора**: `audit_log(actor_id, action, target, created_at)` —
   входы, изменения прав, действия с ПДн; выгрузка для 24-часового
   уведомления РКН при инциденте.
3. **Режим аттестации** для лицензированного ДПО (ПП № 1678): подтверждение
   личности при итоговых тестах — флаг в модуле quizzes.

## 8. Ограничение целевого рынка (важно для продаж)

Государственные и аккредитованные школы в части основных общеобразовательных
программ обязаны использовать только ГИС (ч. 3.1 ст. 16 273-ФЗ) — этот
сегмент закрыт. Целевые сегменты: частные школы (доп. программы), хоумскулинг,
онлайн-школы, репетиторские и учебные центры, СПО/ДПО, вузы (замена Moodle),
корпоративное обучение. Подробности — research-ru-market.md.
