# Модули

Каждый модуль — папка в `server/src/modules/` с Express Router.
Состав приложения собирается в `server/src/index.ts` (одна строка на модуль).
Формат описания: назначение → файлы → данные → API → точки расширения.

---

## core (не модуль — общее ядро)

`server/src/core/`

- `db.ts` — единственная точка доступа к БД (`get/all/run/tx`). Замена СУБД
  выполняется только здесь (см. ИНТЕГРАЦИЯ.md → PostgreSQL).
- `schema.sql` — вся схема данных с комментариями; диалект переносимый.
- `access.ts` — проверки прав: `requireMember`, `requireTeacher`, `studentIds`.
- `errors.ts` — `ApiError` и единый формат ошибок `{ error: { code, message } }`.
- `validate.ts` — валидация входных данных без внешних библиотек.
- `scheduler.ts` — фоновая публикация отложенных постов (опрос каждые 30 с).

## auth — аккаунты и сессии

Файлы: `passwords.ts` (scrypt), `tokens.ts` (HMAC-токены), `middleware.ts`
(`attachUser`, `requireAuth`, `currentUser`), `routes.ts`.

Данные: `users`.

API: `POST /api/auth/register | login | logout | change-password`,
`GET /api/auth/me`, `PATCH /api/auth/profile`.

Расширение: вход через VK ID/Яндекс ID/ЕСИА добавляется новым маршрутом,
который после OAuth-обмена находит/создаёт `users`-запись и вызывает
`issueToken()` — ядро сессий не меняется. Флаг `registration` отключает
самостоятельную регистрацию.

## courses — курсы и участники

Данные: `courses`, `course_members` (роль TEACHER/STUDENT — атрибут связи).

API: CRUD курса, `POST /join` (по коду), `GET /:id/members`, `POST /:id/invite`
(по email с уведомлением), `DELETE /:id/members/:userId` (учитель удаляет /
ученик выходит сам), `POST /:id/archive | restore | copy`, `POST /:id/code/reset`.

Особенности: код приглашения — 7 символов без неоднозначных букв; копирование
курса переносит темы и задания (черновиками), но не учеников и работы;
настройки `stream_mode` (режим ленты) и `grade_scale` (шкала отображения).

## topics — темы (разделы заданий)

Данные: `topics`. API: список/создание в курсе, переименование, удаление
(задания темы не удаляются — остаются «без темы»).

## coursework — задания, вопросы, материалы

Данные: `coursework`, `coursework_assignees`, `rubrics`, `rubric_criteria`,
`rubric_levels`.

API: `GET/POST /api/courses/:id/coursework`, `GET/PATCH/DELETE /api/coursework/:id`,
`POST /:id/publish`, `PUT/DELETE /:id/rubric`.

Логика видимости: ученик видит только PUBLISHED и только адресованные ему
(пустой список назначений = весь курс). Черновики и запланированные видит
только учитель. Публикация рассылает уведомления аудитории.

## quizzes — тесты с автопроверкой

Файлы: `service.ts` (валидация вопросов, автопроверка), `routes.ts`.

Данные: `quiz_questions` (SINGLE/MULTI/TEXT, варианты, правильные ответы,
баллы), `quiz_answers` (ответы и начисленные баллы).

API: `GET/PUT /api/coursework/:id/quiz`. Ученику вопросы отдаются **без
правильных ответов** (`service.studentView`). Сумма баллов вопросов
автоматически становится `max_points` задания.

Автопроверка: при сдаче (`submissions → turn-in`) вызывается `autograde()`;
итог пишется в `draft_grade`. Текстовые ответы нормализуются (регистр,
пробелы, ё→е). Настройка «показывать баллы сразу» — `quiz_show_score`.

## submissions — сдачи и оценивание

Данные: `submissions` (UNIQUE по заданию+ученику), `submission_events`
(append-only история), `submission_rubric_grades`.

API: `GET /api/coursework/:id/submissions` (учитель, создаёт недостающие
записи), `GET /api/coursework/:id/my` (ученик), `GET/PATCH /api/submissions/:id`,
`POST /:id/turn-in | reclaim | grade | return`, `PUT /:id/rubric`.

Жизненный цикл: ASSIGNED → TURNED_IN → RETURNED (оценка опубликована);
RECLAIMED — отмена сдачи учеником; после возврата возможна пересдача.
`allow_late=0` блокирует сдачу после срока. Возврат уведомляет ученика.

## announcements — лента

Данные: `announcements` (+pinned). API: список/создание в курсе,
PATCH/DELETE, `POST /:id/pin`. Права публикации учеников определяет
`courses.stream_mode`. Отложенная публикация — через scheduler.

## comments — комментарии

Данные: `comments` (scope: ANNOUNCEMENT | COURSEWORK | SUBMISSION).
SUBMISSION-ветка приватна: ученик-владелец + учителя курса. Уведомления
участникам ветки. Режим ленты TEACHERS_ONLY запрещает публичные
комментарии учеников.

## grades — журнал

API: `GET /api/courses/:id/grades`. Учитель: ученики × задания + все статусы;
ученик: только свои опубликованные оценки. Шкала отображения — на клиенте
(`formatGrade`), хранение всегда в баллах.

## files — файлы и вложения

Файлы: `attachments.ts` (общая логика вложений), `routes.ts`.

Данные: `attachments` — полиморфные (owner_type: UPLOAD → COURSEWORK /
ANNOUNCEMENT / SUBMISSION).

Протокол: клиент загружает файл (`POST /api/files`) → получает id записи
со статусом UPLOAD → передаёт id при создании поста → запись «закрепляется»
за владельцем. Чужую загрузку прикрепить нельзя. Скачивание
(`GET /api/files/:id`) проверяет права по владельцу вложения.

Расширение: для S3-совместимого хранилища заменяется только storage у multer
и `sendFile` на проксирование/redirect — интерфейс API сохраняется.

## notifications — уведомления

Файлы: `service.ts` (создание + каналы), `routes.ts` («колокольчик»).

Точка расширения — интерфейс `NotificationChannel`:

```ts
import { registerChannel } from './modules/notifications/service.js';

registerChannel({
  deliver(msg) { /* отправка в Telegram/email/SMS */ },
});
```

Внутренний канал (БД) работает всегда; внешние не блокируют основной поток
и не роняют его при ошибках.

## planner — список дел и календарь

API: `GET /api/todo` (toSubmit/done — ученик; toReview — учитель; роли
смешиваются по курсам), `GET /api/calendar?from&to` (дедлайны недели).

## comment-bank — банк комментариев

Личные шаблоны учителя для проверки работ. CRUD `/api/comment-bank`.

# Фронтенд

`web/src/` зеркалит серверные домены:

- `theme.css` — дизайн-токены и все стили (см. WHITELABEL.md);
- `brand.tsx` — загрузка конфига, применение токенов;
- `api.ts` — единый HTTP-клиент; `types.ts` — DTO-типы;
- `components/ui.tsx` — UI-кит (Modal, Menu, Toast, вложения);
- `components/Shell.tsx` — каркас (шапка, боковое меню, уведомления);
- `pages/` — страницы; `pages/course/` — вкладки курса.

Маршруты: `/` (курсы), `/courses/:id` (+`?tab=`), `…/coursework/new|:id|:id/edit|:id/review`,
`/todo`, `/calendar`, `/archive`, `/profile`.
