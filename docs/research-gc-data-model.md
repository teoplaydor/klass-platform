# Отчёт исследования: gc-data-model

Google Classroom API (https://classroom.googleapis.com, v1) построен вокруг иерархии REST-ресурсов с корневой сущностью Course («курс — фундаментальный объект данных»). Вложенные ресурсы курса: aliases, teachers, students, topics, announcements, courseWork (с под-ресурсами studentSubmissions, rubrics, addOnAttachments), courseWorkMaterials, studentGroups (с studentGroupMembers), posts. На верхнем уровне также живут: invitations (приглашения в курс), userProfiles (глобальные профили пользователей с под-ресурсами guardians и guardianInvitations) и registrations (push-уведомления через Cloud Pub/Sub). Все идентификаторы — строки, назначаемые сервером; у курсов дополнительно есть алиасы в двух неймспейсах (домена «d:» и проекта разработчика «p:»). Времена — RFC 3339 Timestamp, даты дедлайнов разделены на пару Date (UTC) + TimeOfDay (UTC).

Лента курса (stream) состоит из трёх типов постов: CourseWork (задания, принимающие сдачи и оценки), CourseWorkMaterial (материалы без сдачи) и Announcement (объявления). Все три имеют общий набор полей: courseId, id, state (PUBLISHED/DRAFT/DELETED, по умолчанию DRAFT), creationTime/updateTime, scheduledTime (отложенная публикация), creatorUserId, alternateLink, materials[] (до 20 вложений), assigneeMode (ALL_STUDENTS | INDIVIDUAL_STUDENTS) + individualStudentsOptions (список studentIds). CourseWork и CourseWorkMaterial дополнительно ссылаются на Topic через topicId. Material — это union-тип вложения: SharedDriveFile (DriveFile + shareMode: VIEW/EDIT/STUDENT_COPY), YouTubeVideo, Link, Form (read-only), а также новые read-only типы GeminiGem и NotebookLmNotebook; у всех вложений типовой набор полей id/url, title, alternateLink, thumbnailUrl.

Жизненный цикл сдачи: StudentSubmission создаётся автоматически для каждого студента при публикации CourseWork (одна сдача на пару courseWorkId+userId). Состояния (SubmissionState): NEW/CREATED → TURNED_IN → RETURNED, плюс RECLAIMED_BY_STUDENT (студент отозвал сдачу) и STUDENT_EDITED_AFTER_TURN_IN; переходы выполняются методами turnIn/return/reclaim. Контент сдачи — union по типу работы: assignmentSubmission (attachments[]), shortAnswerSubmission (answer), multipleChoiceSubmission (answer). Оценки двухфазные: draftGrade (черновик, виден только учителям) → assignedGrade (выставляется при возврате; не может существовать без draftGrade); при наличии рубрики аналогично draftRubricGrades/assignedRubricGrades (map criterionId → {levelId, points}). Вся история изменений хранится в submissionHistory[] — union из StateHistory {state, stateTimestamp, actorUserId} и GradeHistory {pointsEarned, maxPoints, gradeTimestamp, actorUserId, gradeChangeType}. Это даёт готовый паттерн для БД: таблица сдач + append-only таблица событий истории.

Роли и люди: UserProfile — глобальная сущность (id, name{given/family/full}, emailAddress, photoUrl, permissions[CREATE_COURSE], verifiedTeacher); Teacher и Student — это связки «membership» курс↔пользователь (courseId + userId + встроенный profile), т.е. роли локальны для курса, а не глобальны. Вступление в курс идёт через Invitation {id, courseId, userId, role: STUDENT/TEACHER/OWNER} с методом accept либо по enrollmentCode. Guardian {studentId, guardianId, guardianProfile, invitedEmailAddress} — связь «опекун↔студент», создаваемая через GuardianInvitation {studentId, invitationId, invitedEmailAddress, state: PENDING/COMPLETE}. Оценочная модель курса: gradebookSettings {calculationType: TOTAL_POINTS|WEIGHTED_CATEGORIES, displaySetting, gradeCategories[]}, GradeCategory {id, name, weight (целое, 12.34% = 123400), defaultGradeDenominator}, и GradingPeriodSettings {gradingPeriods[{id, title, startDate, endDate}] — без пересечений дат, applyToExistingCoursework} с привязкой CourseWork.gradingPeriodId. Add-on слой: AddOnAttachment {courseId, itemId (id поста любого из трёх типов), id, title, teacherViewUri/studentViewUri/studentWorkReviewUri (EmbedUri для iframe), dueDate/dueTime, maxPoints, copyHistory[]} плюс AddOnAttachmentStudentSubmission {id, userId, courseWorkSubmissionId, pointsEarned, postSubmissionState} для оценок за вложение (grade passback).

## Факты
- Базовый endpoint API: https://classroom.googleapis.com (v1); иерархия ресурсов: courses -> {aliases, teachers, students, topics, announcements, courseWork -> {studentSubmissions, rubrics, addOnAttachments -> studentSubmissions}, courseWorkMaterials, studentGroups -> studentGroupMembers, posts}; на верхнем уровне: invitations, registrations, userProfiles -> {guardians, guardianInvitations}.
- Course: поля id (read-only, назначается сервером, при создании можно передать алиас), name (обязательное, 1-750 символов), section (до 2800), descriptionHeading (до 3600), description (до 30000), room (до 650), ownerId (обязателен при создании; числовой id, email или 'me'), subject, levels, creationTime, updateTime, enrollmentCode (read-only, код для самостоятельного вступления), courseState, alternateLink, teacherGroupEmail, courseGroupEmail, teacherFolder (DriveFolder), guardiansEnabled, calendarId, gradebookSettings, courseMaterialSets[] (deprecated).
- CourseState (enum): ACTIVE, ARCHIVED (модификация запрещена кроме смены состояния), PROVISIONED (создан, не активирован — значение по умолчанию), DECLINED, SUSPENDED; плюс служебное COURSE_STATE_UNSPECIFIED.
- Course.gradebookSettings: calculationType (TOTAL_POINTS | WEIGHTED_CATEGORIES), displaySetting (SHOW_OVERALL_GRADE | HIDE_OVERALL_GRADE | SHOW_TEACHERS_ONLY), gradeCategories[]; GradeCategory = {id, name, weight: integer с двумя знаками точности (123400 = 12.34%, только для WEIGHTED_CATEGORIES), defaultGradeDenominator (только для TOTAL_POINTS)}.
- CourseWork: courseId, id (уникален в пределах курса), title (обязателен, 1-3000 символов), description (до 30000), materials[] (до 20), state (CourseWorkState: PUBLISHED/DRAFT/DELETED, default DRAFT; DRAFT и DELETED видны только учителям и админам), alternateLink, creationTime, updateTime, dueDate (тип Date, UTC) + dueTime (TimeOfDay, UTC — задаются только парой), scheduledTime, maxPoints (0 или null = неоцениваемое задание), workType (immutable после создания), assignment.studentWorkFolder (DriveFolder, только при workType=ASSIGNMENT), multipleChoiceQuestion.choices[] (только при MULTIPLE_CHOICE_QUESTION), assigneeMode, individualStudentsOptions.studentIds[], submissionModificationMode (MODIFIABLE_UNTIL_TURNED_IN default | MODIFIABLE), creatorUserId, topicId (FK на Topic курса), gradeCategory, gradingPeriodId, associatedWithDeveloper.
- CourseWorkType (enum): ASSIGNMENT, SHORT_ANSWER_QUESTION, MULTIPLE_CHOICE_QUESTION (+ COURSE_WORK_TYPE_UNSPECIFIED, никогда не возвращается); тип задаётся при создании и не меняется.
- AssigneeMode (общий enum для CourseWork/CourseWorkMaterial/Announcement): ALL_STUDENTS (default) | INDIVIDUAL_STUDENTS; при INDIVIDUAL_STUDENTS заполняется individualStudentsOptions со списком studentIds — в БД это таблица связи пост-студент.
- CourseWorkMaterial: подмножество полей CourseWork без оценивания и сдач — courseId, id, title, description, materials[], state (CourseWorkMaterialState: PUBLISHED/DRAFT/DELETED), alternateLink, creationTime, updateTime, scheduledTime, assigneeMode, individualStudentsOptions, creatorUserId, topicId; нет dueDate, maxPoints, workType, submissions.
- Announcement: courseId, id, text (до 30000 символов — вместо title/description), materials[], state (AnnouncementState: PUBLISHED/DRAFT/DELETED), alternateLink, creationTime, updateTime, scheduledTime, assigneeMode, individualStudentsOptions, creatorUserId; нет topicId, дедлайнов и сдач.
- Material — union-тип вложения (ровно одно из): driveFile (SharedDriveFile = {driveFile: DriveFile, shareMode}), youtubeVideo, link, form (read-only при создании), gem (GeminiGem, read-only), notebook (NotebookLmNotebook, read-only); ShareMode: VIEW (default) | EDIT | STUDENT_COPY (личная копия каждому студенту).
- Типовые поля вложений: DriveFile {id (Drive API resource id), title, alternateLink, thumbnailUrl}; YouTubeVideo {id, title, alternateLink, thumbnailUrl}; Link {url, title, thumbnailUrl — title и thumbnail Classroom получает сам}; Form {formUrl, responseUrl (только для редакторов формы при наличии ответов), title, thumbnailUrl}; GeminiGem/NotebookLmNotebook {id, title, url}.
- StudentSubmission: создаётся автоматически для каждого назначенного студента при создании CourseWork; уникальна для пары courseWorkId+userId; поля: courseId, courseWorkId, id, userId, creationTime/updateTime (могут быть не заданы, пока студент не открыл задание), state, late (boolean), draftGrade, assignedGrade, rubricId, draftRubricGrades, assignedRubricGrades, alternateLink, courseWorkType, associatedWithDeveloper, submissionHistory[], плюс union-контент: assignmentSubmission.attachments[] | shortAnswerSubmission.answer | multipleChoiceSubmission.answer.
- SubmissionState (enum): NEW (студент ещё не открывал; в актуальной документации стартовое состояние обозначено как CREATED, NEW встречается в API как 'never accessed'), CREATED, TURNED_IN, RETURNED, RECLAIMED_BY_STUDENT (студент отозвал сдачу), STUDENT_EDITED_AFTER_TURN_IN; переходы через методы turnIn, return, reclaim; modifyAttachments добавляет вложения.
- Двухфазная модель оценок: draftGrade (черновая, видна только учителям, неотрицательная, до 2 знаков после запятой) -> assignedGrade (выставляется при возврате работы, в UI автоматически копируется из draftGrade; не может существовать без draftGrade); методы смены состояния (return) сами оценку НЕ ставят — нужен отдельный patch с updateMask 'draftGrade,assignedGrade'.
- submissionHistory[] — append-only история сдачи, union двух типов записей: StateHistory {state, stateTimestamp, actorUserId (кто сменил состояние — учитель или студент)} и GradeHistory {pointsEarned (числитель), maxPoints (знаменатель на момент оценки), gradeTimestamp, actorUserId, gradeChangeType: DRAFT_GRADE_POINTS_EARNED_CHANGE | ASSIGNED_GRADE_POINTS_EARNED_CHANGE | MAX_POINTS_CHANGE}; статусы missing/complete/excused в API отдельно не выражены.
- Rubric (под-ресурс courseWork, max 1 на задание): {courseId, courseWorkId, id, creationTime, updateTime, criteria[], sourceSpreadsheetId (input-only, импорт из Google Sheets)}; Criterion = {id, title, description, levels[]}; Level = {id, title, description, points (опционально; если заданы — все уровни критерия должны иметь различные значения)}; оценки по рубрике в сдаче — map criterionId -> RubricGrade {criterionId, levelId, points}.
- Topic: {courseId, topicId, name (непустое, до 100 символов, уникальность по имени в курсе), updateTime}; служит только для группировки CourseWork и CourseWorkMaterial через их topicId.
- UserProfile (глобальный пользователь): {id, name {givenName, familyName, fullName}, emailAddress (требует scope profile.emails), photoUrl (scope profile.photos), permissions[] (enum: CREATE_COURSE), verifiedTeacher (boolean)}; все поля read-only.
- Teacher = membership {courseId, userId, profile (embedded UserProfile)}; Student = membership {courseId, userId, profile, studentWorkFolder (DriveFolder, виден только студенту и админам)}; роли — атрибут связи курс-пользователь, один пользователь может быть учителем в одном курсе и студентом в другом; в БД это две таблицы связи (или одна с полем role).
- Invitation: {id, courseId, userId, role (CourseRole: STUDENT | TEACHER | OWNER)}; принимается методом accept, после чего создаётся membership; альтернативный путь вступления студента — enrollmentCode курса.
- Guardian: {studentId, guardianId, guardianProfile (UserProfile), invitedEmailAddress (видно только админам домена)} — many-to-many связь опекун-студент; создаётся только через GuardianInvitation {studentId, invitationId, invitedEmailAddress, state: PENDING | COMPLETE, creationTime}; у курса есть флаг guardiansEnabled.
- GradingPeriodSettings (настройка курса): {gradingPeriods[], applyToExistingCoursework (персистентный boolean — авто-раскладка существующих заданий по периодам)}; GradingPeriod = {id (read-only), title (уникален в курсе), startDate, endDate (Date, UTC, включительно)}; периоды не должны пересекаться и идут хронологически; связь с заданием через CourseWork.gradingPeriodId (по умолчанию вычисляется из dueDate).
- AddOnAttachment (вложение стороннего приложения к любому из трёх типов постов): {courseId, itemId (id поста; postId deprecated), id (уникален в пределах поста), title (1-1000), teacherViewUri/studentViewUri (обязательные EmbedUri для iframe), studentWorkReviewUri (опционально; без него нельзя grade passback), dueDate+dueTime, maxPoints, copyHistory[] (цепочка копий при копировании курса)}; itemType контекста: courseWork | courseWorkMaterials | announcement.
- AddOnAttachmentStudentSubmission: {id, userId, courseWorkSubmissionId (FK на основную StudentSubmission), pointsEarned (оценка за вложение), postSubmissionState (состояние родительской сдачи)} — механизм передачи оценок (grade passback) из add-on в Classroom.
- StudentGroup: {courseId, id, title} с членами через courses.studentGroups.studentGroupMembers (create/delete/list) — группировка студентов внутри курса.
- Registration (push-уведомления): подписка фида на Cloud Pub/Sub topic; типы фидов: COURSE_ROSTER_CHANGES (на домен) и COURSE_WORK_CHANGES (на курс — изменения courseWork и studentSubmissions); регистрация живёт неделю и продлевается повторным create.
- Алиасы курса (courses.aliases): альтернативные идентификаторы в двух неймспейсах — доменном ('d:', создаёт админ, виден всему домену) и проектном ('p:', привязан к проекту разработчика); полезный паттерн для таблицы внешних ключей-синонимов.
- Сквозные паттерны схемы для проектирования БД: (1) все id — строки, генерируемые сервером, уникальные в пределах родителя; (2) у всех контентных сущностей creationTime/updateTime + state-enum с мягким удалением (DELETED); (3) денормализация courseId во все вложенные сущности; (4) полиморфные вложения через union-тип Material; (5) history как append-only лог событий с actorUserId; (6) двухфазность draft/assigned для оценок; (7) видимость полей зависит от роли (teacherFolder — учителям, studentWorkFolder — студенту, invitedEmailAddress — админам).
